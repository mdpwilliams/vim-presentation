vim

When you are writing code in the editor, how do you spend time in the editor?

I don't know about you're work flow specifically, but I spend more time searching and researching a solution rather than implementing it.

Writing code is not like writing creatively, it is more akin to tuning an engine than reading a book.

vim intends to switch the priority from entering code, to navigating and modifying code to increase speed and efficiency.

Opening/Closing Files

If you are using a plugin for your editor of choice, stick with their method of opening files.

To open a file, using the terminal you need a command like:

vim path/to/file.txt

Exiting vim is the greatest escape room.

Closing a file

While in vim, there are multiple ways to exit:

:wq
:x
ZZ
:q!
:qa

Most of these are entered in command mode. The colon signifies that you are entering a Command Mode command.

  • Write current file and exit.
  • Exit vim, but write only when changes have been made.
  • Equivalent to 
    :x
    . This is a keypress.
  • Exit Vim without saving currently opened file.
  • Exit all open files in current Vim session.

Modes

There are 5 modes in vim

  • Normal Mode
  • Insert Mode
  • Command Mode
  • Visual Mode
  • Insert Normal Mode

There are a couple other modes like visual-block mode, replace mode

.

Realistically, I would only be concerned with the first four to get the most out of your vim learning

.

Normal Mode

This is the mode entered in by default.

The keyboard is mapped to an assortment of different commands that allow you to navigation and manipulate text.

The reason people struggle to learn vim is that the editor doesn't naturally default to text editting by default.

You should always be trying to get back to this mode.

Insert Mode

This is intended for "inserting" text into the document.

From normal mode, you can jump in at the cursor using 'a' or 'i', to "append" text after or "insert" text before.

Does what it says on the tin. Your keyboard should function as expected with the exception of a couple commands

Visual Mode

For navigation and text manipulation.

Entering this mode allows you to move around like within normal mode, but movement extends a highlighted area to target text.

Command Mode

Meant for running Ex commands, searches, or filter commands.

An Ex command usually begins: 

:{command}

For example, exiting vim is an Ex command: 

:q

Navigation

vim's ability to put code navigation on the keyboard, in easy to reach places is what helps you improve your speed.

Basic Movement

Up, down, left and right are just off the home row.

  • 'h', left
  • 'j', down
  • 'k', up
  • 'l', right

You can use the arrow keys, but I highly suggest against this.

Word Movement

Instead of moving on a character basis, moving by word is far faster and far more accurate.

  • 'w', start of the next word
  • 'W', start of the next WORD
  • 'e', end of the next word
  • 'E', end of the next WORD
  • 'b', Go to the word before
  • 'B', Go to the WORD before

Difference between word and WORD is what are breaking characters.

Breaking characters are things like brackets, parantheses, punctuation

For WORDS, it ignores breaking characters and uses strictly whitespace.

Scrolling

When working with larger files, it will be easier to move through the file in a page-by-page manner

  • 'ctrl-d', scroll down half page
  • 'ctrl-u', scroll up half page
  • 'ctrl-f', scroll down full page (or forwards)
  • 'ctrl-b', scroll up full page (to beginning, or backwards)

Jumping

vim also give you the ability to jump through huge sections of code, or onto specific lines.

  • 'gg' Goes to the first line of the file
  • 'G' Goes to the bottom line of the file
  • '{ }' Goes the beginning or ending of the paragraph
  • '%' Go to the matching ( ), [ ], { }
  • '50%' Go to the line at the 50% mark of the file
  • ':NUM' Go to the specific line number.

Search

Basic Search

You can search forward by '/' and entering your search pattern.

Like, '/code' will search for all matches of code in your file.

Clicking enter will search for that search pattern and highlight all matches, in addition to jumping you to the next one.

Afterwards 'n' and 'N' will move you through the search results forward and backwards, respectively

Some notes about search...

This is a pattern, you can use regex to make a more complex search criteria

You can also search in reverse using '?' this reverses 'n' and 'N'

Complex patterns, like adding ORs or wild characters

Super advanced tip: vim remembers where you were when you wanted to first searched, after you completed your search if you press 'ctrl-o' it will jump back to your previous position. (You can use 'ctrl-i' to jump forward in positions)

Searching by Word

If you want to find the word under the cursor, you can use

'*' or '#'

These work both forwards and backwards, respectively

Please note that these are word specific, and not WORD; so they break on breaking characters like brackets, commas, etc.

Undo and Redo

Before we get into making sweeping changes, we're quickly going to talk about undoing and redoing changes.

  • 'u', is undo
  • 'ctrl-r', is redo

Like verbs that we'll learn about later, this can take numerical modifiers.

Vim also has commands that let you rewind changes over time.

Let's say you hypothetically let your corgi type on your computer for ten minutes, destroying your presentation!

:earlier 10m

Or let's say the same Corgi solved a bug you just rewinded!

:later 10m

I'll be honest, I haven't had the opportunity to use this a ton. I can see how it would be helpful, just haven't had the chance.

Verbs

Powerless Verbs

Actions that do not take a modifer, and run a quick action

  • 'x', deletes the character underneath the cursor to the right
  • 'X', deletes the character underneath the cursor to the left
  • 'r', enters replace mode, replace character under cursor with a new character
  • 's', delete the character under the cursor and enter Insert mode

Quick actions, used for quick editting, note that the deletes are stored in the clipboard, so let's say you write two letters in the incorrect order you could use 'xp' to correct them.

Powerful Verbs

These actions are more powerful, usually taking a modifier that allows them to target sections of code.

  • 'y', yank (copy)
  • 'c', change
  • 'd', delete (cut)
  • 'v', visually-select

Mitch? Why did you add 'v'! I added that because it works with modifiers and then used with verbs. It's kind of like an adverb!

Also interesting! Pressing the command again after entering it will default to running the command on the current line you're on.

Modifiers and Nouns

Modifiers

Used right before nouns, modifiers influence how the noun is going to be effected.

  • 'i', for inner (or inside)
  • 'a', for around (or all)
  • NUM, a number
  • 't, T', search for a character and stop before (until)
  • 'f, F', search for a character and the searched character (find)
  • '/', next searched pattern

These are a little tough without context but these are helping you restrict and focus the effects of the powerful verbs

not that the 'T' and 'F' work in the reverse direction.

Nouns

Just like English, nouns are objects that you apply effects to.

  • 'w,W', start of next word or WORD
  • 'e,E', end of word or WORD
  • 'b,B', start of previous word or WORD
  • 's', sentence
  • 'p', paragraph
  • 't', tag (with respect to HTML)
  • 'b', block (with respect to programming)
  • 'h j k l', for a direction
  • '$', to the end of the line
  • '^', to the beginning of the line

All Together Now!

Using powerful verbs in tandem with modifiers and nouns you can create commands that quickly structure changes you want to make.

console.log('Data received');

Search and Replace

This is where we start putting it all together.

We know that the command for substitution is 's'.

:[range] s/[pattern]/[string]/[flags] [count]

We already know most of these. Pattern, string, count.

Ranges

For most commands, the target is the line.

When we apply a command we can modify the range of its effect.

  • NUM, apply to a specific line
  • 1, first line
  • $, last line
  • %, all lines
  • ., current line
  • 50,$ , line fifty to the end of the file
  • <, > , use the visual mode selection

Flags

  • g, change will be performed over all occurrences
  • i, pattern is case insensitive, 'home' is the same as 'HOME'
  • c, runs interactively
Interactive search and replace runs through the matches and let's you choose whether the replacement takes place or not.
